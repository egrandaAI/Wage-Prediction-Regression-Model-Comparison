Executive Summary

What matters most for a decision-maker

Business question

Can wage levels be predicted with enough reliability to support compensation benchmarking and workforce analytics, and which regression approach offers the best balance between accuracy and generalization?

Main conclusion

The project shows that wage prediction is feasible, but only with moderate explanatory power. More complex models improved fit in some cases, yet also increased overfitting risk. The final choice therefore favored robustness over apparent training performance.

Strategic takeaway

The dataset supports directional wage insight rather than high-stakes salary decisions. It is useful for exploratory benchmarking, but not sufficient on its own for production-grade pay recommendations without richer labor market variables.

Problem Context

Why this project is relevant

Why wage prediction matters

  • Supports salary benchmarking and compensation analysis.
  • Helps identify structural drivers of income disparities.
  • Enables more evidence-based workforce and HR analytics.
  • Creates a foundation for future interactive decision tools.

Project objective

Build a structured regression pipeline that handles preprocessing, encoding, feature selection, model comparison, and validation, then select the model with the strongest cross-validation performance and assess residuals and feature importance.

Analytical Approach

How the notebook was translated into a rigorous workflow

1. Data preparation

The workflow addressed missing values, converted object variables into numeric form where needed, and applied one-hot encoding to non-ordinal categorical fields such as race.

2. Exploratory analysis

Numerical and categorical distributions were reviewed to understand earnings patterns and feature behavior. Categories with very similar earnings distributions were consolidated to improve interpretability.

3. Model evaluation

Several regressors were compared through training/test performance and grid search. Cross-validation was used as the main decision criterion to avoid choosing an overfit model.

Key Findings

Insights emerging from the notebook narrative and outputs

Model behavior

  • Random Forest achieved very high in-sample fit but deteriorated sharply on the test set, signaling clear overfitting.
  • Gradient Boosting improved test performance and reduced the overfit gap, making it the strongest manual model among the tested nonlinear approaches.
  • Grid search ultimately selected Ridge(alpha = 10), indicating that regularization offered the best balance of stability and out-of-sample consistency in the comparison framework.

Data interpretation

  • Wage patterns appear only partially explained by the available variables.
  • Categorical socioeconomic features matter, but the current feature space still leaves substantial unexplained variance.
  • The notebook’s own commentary correctly recognizes that the best scores remain useful but not fully satisfactory, which is an important sign of methodological honesty.

Recommendation

What should be taken forward

Recommended model for the project deliverable

Ridge Regression (alpha = 10) should remain the final selected model because the project brief explicitly prioritizes the highest cross-validation performer. Even if Gradient Boosting delivered a higher test R² in one split, Ridge is the more defensible choice under the assignment’s generalization-first logic.

How to position the result professionally

This project should be framed as a strong example of responsible model selection: the final recommendation was not based on the most visually appealing training score, but on the model that best balanced robustness, reproducibility, and methodological rigor.

Business Implications

Where this analysis is useful in practice

Compensation benchmarking

Useful for directional salary analysis, identifying broad income patterns, and testing how demographic or professional features relate to wage outcomes.

HR analytics

Can support exploratory fairness reviews and pay-structure analysis, especially when paired with richer contextual variables such as geography, firm size, or tenure.

Decision support

Suitable as an analytical layer, not as an autonomous wage-setting engine. Human review and richer labor market data remain essential.

Limitations & Next Steps

What would strengthen the model

Current limitations

  • Moderate predictive power suggests omitted variables are still significant.
  • Some nonlinear models improved fit but at the cost of generalization.
  • The available data appears better suited for explanatory analysis than highly accurate wage prediction.

Recommended next steps

  • Add richer variables such as location, sector, firm size, tenure, and experience depth.
  • Test additional regularized and boosting approaches with stronger feature engineering.
  • Introduce SHAP or similar explainability techniques for a more executive-friendly interpretation layer.
  • Convert the final model into an interactive benchmarking tool only after improving the feature space.
